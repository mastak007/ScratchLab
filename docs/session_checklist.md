# Session Checklist

## Before The Session

- [ ] Confirm DJ name for the session folder
- [ ] Confirm session date
- [ ] Create the session folder
- [ ] Confirm scratch type is `baby` only
- [ ] Confirm BPM targets are `70`, `90`, and `110`
- [ ] Mount the primary iPhone so the hands, platter, and fader are visible
- [ ] Open Serato recording and confirm WAV capture
- [ ] If using a second phone, label it as `camB`
- [ ] If using an Apple Watch, wear it on the scratching hand and open the capture app
- [ ] Clear enough local storage on every recording device

## Before Each Take

- [ ] Select the target BPM
- [ ] Confirm the take number for that BPM
- [ ] Start recording on each active device
- [ ] Get hands into frame
- [ ] Keep background talking to a minimum

## Take Script

- [ ] Say: `baby scratch, [BPM], take [number]`
- [ ] Clap three times
- [ ] Perform scratch 1 for about 20 seconds
- [ ] Clap once
- [ ] Perform scratch 2 for about 20 seconds
- [ ] Clap once
- [ ] Perform scratch 3 for about 20 seconds
- [ ] Stop all recordings

## Quick Quality Check

- [ ] Verbal slate is audible
- [ ] Triple clap is visible and audible
- [ ] Separator claps are present
- [ ] There are exactly three scratch sections
- [ ] The scratch type stayed baby scratch only
- [ ] The BPM matches the slate
- [ ] Primary `camA` view is usable
- [ ] Serato audio file exists
- [ ] Watch file exists or was intentionally skipped

## After Each Take

- [ ] Copy original files into `raw/`
- [ ] Keep a note if anything unusual happened
- [ ] Retake immediately if the take is not usable

## End Of Session

- [ ] At least one valid take exists at `70` BPM
- [ ] At least one valid take exists at `90` BPM
- [ ] At least one valid take exists at `110` BPM
- [ ] Raw files are all in the session folder
- [ ] `manifests/take_log.csv` has one row for every renamed take
- [ ] Files are renamed to the standard format
- [ ] Session manifest and take log are saved in `manifests/`
- [ ] Session validation has been run
- [ ] Missing or invalid items are written down before the session is closed
